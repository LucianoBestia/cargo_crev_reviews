Integrity checks

When we are reviewing the source code of a crate, we must be sure that it is not compromised.
The commands `cargo check` or `cargo build` download files with the `.crate` extension from crates.io to the `cargo registry cache` for all the crates in your project dependency tree.
It then checks the cksum field from the `cargo registry index` database to test the integrity of the downloaded files.
Then it unpacks this files into the `cargo registry src` directory. This is the original source code that cargo uses to compile into your project.
Sometimes it can happen that this source code is altered - unknowingly or unwillingly. Most of the time by the intellisense server of the code editor.
Use the command `List unclean crates` to get a list of crates to manually delete from the `src` folder.

If the crate was unclean when writing the crev review, the calculated digest is incorrect in the saved review.
You can check and correct these incorrect digests.
